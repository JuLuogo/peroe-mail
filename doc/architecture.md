## 请求处理流程

### 1. 用户请求流程

```
用户浏览器
    │
    ▼ HTTPS 请求
Cloudflare CDN
    │
    ├── 静态资源请求 (/static/*, /attachments/*)
    │       │
    │       ▼
    │   直接返回静态文件
    │
    └── API 请求 (/api/*)
            │
            ▼
    路由到 mail-worker
            │
            ▼
    JWT 认证中间件
            │
            ├── 失败 → 返回 401
            │
            └── 成功
                    │
                    ▼
            权限检查中间件
                    │
                    ├── 无权限 → 返回 403
                    │
                    └── 有权限
                            │
                            ▼
                    API 路由处理
                            │
                            ▼
                    返回 JSON 响应
```

### 2. 邮件接收流程

```
外部邮件服务器
    │
    ▼ SMTP
Cloudflare 邮件路由
    │
    ▼ MX 记录
mail-worker.email() 函数
    │
    ▼
PostalMime 解析邮件
    │
    ▼
查找收件人账号
    │
    ├── 未找到 → 拒绝邮件
    │
    └── 找到
            │
            ▼
    检查 Catch-all 规则
            │
            ▼
    权限验证（域名使用权限）
            │
            ▼
    保存邮件到 D1 (状态: SAVING)
            │
            ▼
    保存附件到 R2
            │
            ▼
    更新邮件状态 (RECEIVE / NOONE)
            │
            ▼
    发送 Telegram 通知（如配置）
            │
            ▼
    执行转发规则（如配置）
```

### 3. 邮件发送流程（同步）

```
前端提交邮件
    │
    ▼ POST /api/email/send
权限检查
    │
    ▼
判断收件人类型
    │
    ├── 全为站内用户
    │       │
    │       ▼
    │   直接保存到 D1
    │
    └── 包含站外邮箱
            │
            ▼
    选择发送方式
    │
    ├── Resend → 调用 Resend API
    ├── SES → 调用 AWS SES
    └── Queue → 发送到队列
            │
            ▼
    保存附件到 R2
            │
            ▼
    更新发送计数
            │
            ▼
    返回结果
```

### 4. 邮件发送流程（异步队列）

```
前端提交邮件
    │
    ▼ POST /api/email/send
权限检查
    │
    ▼
入队到 Cloudflare Queue
    │
    ▼
立即返回 "queued" 状态
    │
    ▼
Queue Consumer 处理
    │
    ▼
调用本地 SES API 或 Resend
    │
    ▼
更新邮件状态
```

## 数据流架构

### 用户数据流

```
注册/登录
    │
    ▼
前端 POST /api/login 或 /api/register
    │
    ▼
后端验证 → 生成 JWT Token
    │
    ▼
Token 存储到前端 localStorage
    │
    ▼
后续请求携带 Token
    │
    ▼
后端验证 Token → 获取用户信息
```

### 邮件数据流

```
邮件创建
    │
    ├── 发件：前端 → API → D1
    │
    └── 收件：SMTP → Worker → D1
            │
            ▼
    附件存储
            │
            ▼
    R2 存储附件文件
            │
            ▼
    D1 存储附件元数据
            │
            ▼
    显示邮件列表
            │
            ▼
    用户查看邮件详情
            │
            ▼
    加载附件（如需要）
            │
            ▼
    R2 下载附件
```

## 安全架构

### 多层防护

```
┌─────────────────────────────────────────────┐
│               第一层：网络层                 │

                    │
                    ▼
┌─────────────────────────────────────────────┐
│               第二层：认证层                 │
│  - JWT Token 验证                           │
│  - Token 有效期管理                         │
│  - 多 Token 并发控制                        │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│               第三层：权限层                 │
│  - RBAC 权限控制                            │
│  - API 级别权限检查                         │
│  - 管理员特殊权限                           │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│               第四层：业务层                 │
│  - 输入验证                                 │

```

### 数据安全

| 安全措施 | 说明 |
|----------|------|
| 密码哈希 | PBKDF2 + 随机盐值 |
| JWT 签名 | HMAC-SHA256 |
| HTTPS | 全站强制 HTTPS |
| CORS | 限制跨域访问 |
| 输入验证 | 参数校验和过滤 |
| SQL 注入防护 | Drizzle ORM 参数化查询 |

## 存储架构

### D1 数据库使用

| 表类型 | 用途 | 数据量预估 |
|--------|------|-----------|
| 用户表 | 用户账户信息 | 小 |
| 邮件表 | 邮件内容 | 大 |
| 附件表 | 附件元数据 | 大 |
| 角色表 | 权限配置 | 小 |
| 设置表 | 系统配置 | 小 |

### KV 存储使用

| Key 前缀 | 用途 | 说明 |
|----------|------|------|
| `AUTH_INFO` | 用户认证信息 | Token 列表 |
| `PUBLIC_KEY` | 公开访问 Token | 公开 API 认证 |
| `DAY_SEND_COUNT` | 每日发送计数 | 频率限制 |
| `SETTING` | 系统设置缓存 | 快速读取 |

### R2 存储使用

| 路径前缀 | 用途 | 说明 |
|----------|------|------|
| `attachments/` | 邮件附件 | 用户上传的附件 |
| `background/` | 背景图片 | 登录背景图 |
| `temp/` | 临时文件 | 临时缓存 |

### 状态管理

使用 Pinia 进行状态管理，支持持久化：

```javascript
// store/user.js
export const useUserStore = defineStore('user', {
    state: () => ({
        user: null,
        perms: []
    }),
    actions: {
        async fetchUser() {
            // 获取用户信息
        }
    },
    persist: true  // 持久化到 localStorage
});
```

### 路由守卫

```javascript
// 路由守卫逻辑
router.beforeEach(async (to, from, next) => {
    // 1. 检查是否需要登录
    if (to.meta.requiresAuth && !hasToken()) {
        return next('/login');
    }

    // 2. 检查权限
    if (to.meta.perm && !hasPerm(to.meta.perm)) {
        return next('/403');
    }

    next();
});
```

## 定时任务

### Cron 任务配置

```toml
# wrangler.toml
[triggers]
crons = ["0 16 * * *"]  # 每天 UTC 16:00 (北京时间 00:00)
```

### 任务列表

| 任务 | 执行时间 | 说明 |
|------|----------|------|
| 清理验证码 | 每日 | 删除过期的验证码记录 |
| 重置发送计数 | 每日 | 重置用户每日发送次数 |
| 完成邮件接收 | 每日 | 完成状态异常的邮件 |
| 清理 OAuth 用户 | 每日 | 删除未绑定账号的 OAuth 用户 |
| 清理临时文件 | 每日 | 删除过期的临时文件 |
| 清理过期规则 | 每日 | 删除过期的过滤/转发规则 |
| 清理 TG 归档 | 每日 | 删除过期的 Telegram 归档 |

## 扩展性设计

### 水平扩展

Cloudflare Workers 自动扩展，无需手动配置。

### 垂直扩展

| 限制 | 当前配置 | 扩展方式 |
|------|----------|----------|
| CPU 时间 | 50ms (免费) / 30s (付费) | 升级到付费计划 |
| 内存 | 128MB (免费) / 3000MB (付费) | 升级到付费计划 |
| 请求大小 | 100KB (免费) / 25MB (付费) | 升级到付费计划 |

### 插件化设计

新增功能可以通过以下方式：

1. **新增 API 路由**: 在 `api/` 目录添加
2. **新增 Service**: 在 `service/` 目录添加
3. **新增数据库表**: 在 `entity/` 目录添加
4. **新增前端页面**: 在 `views/` 目录添加

## 容灾设计

### 数据备份

- D1 数据库自动每日备份
- R2 对象存储自动版本控制（可选）

### 故障恢复

| 故障场景 | 恢复方式 |
|----------|----------|
| Worker 崩溃 | Cloudflare 自动重启 |
| D1 不可用 | 使用 KV 缓存降级 |
| R2 不可用 | 返回错误信息 |
